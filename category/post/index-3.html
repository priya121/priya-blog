  1. Loops in Java

    08 September 2015

    Loops in Java are useful even in the most simple program and contain their contents within curly braces. The two most common ones I have used so far are the for loop;

    eg. for( int i = 0; i < 5; i++ )

    and the while loop;

    eg. while ( i < number.length )

    Keeping track of the variables and loops can again be clarified through the use of an instruction table.

    It is also possible to escape out of a loop if a certain condition is met using a label just before the beginning of the loop and a break after the end of the loop.

  2. UML Diagrams

    09 September 2015

    UML diagrams are commonly used when modelling for object oriented design programs. They are the graphical notation for drawing diagrams of software concepts.

    Composite relationships are shown in a UML diagram by an arrow with one end a point and the other a diamond.

    Inheritance is shown through a blank arrowhead and the arrows point in the direction that the inheritance flows from.

    Class icons are divided into three components, a class title, the variables and the functions and interfaces are shown by the <> notation.

    It is always a good idea to draw up a UML before starting to write a program, and adjust it as you go.

  3. Interfaces

    10 September 2015

    Interfaces are a set of classes that used to create an abstraction between two other related classes. They do not contain any new code but hold behaviours and method signatures and should be abstract classes with abstract names.

    The advantage of using interfaces is that they help eliminate unnecessary dependencies that create rigid and fragile software. Adding interfaces hides the implementation of certain classes from others when they are not needed and allows for easier changes to the code later down the line.

    Ruby and other dynamic languages do not need to have an explicit interface class, but it is necessary to define an interface class type explicitly for statically typed languages like Java.

  4. Arrays vs ArrayLists in Java

    11 September 2015

    Difference between an Array List and Array in Java

    In Java Arrays and ArrayLists are differentiated in some important aspects:

    1. An Array is fixed in length, whereas the ArrayList can be added to and is dynamic in size.

    2. ArrayLists cannot contain primitive data types whereas an Array can contain both Object references and primitive types.

    3. .iterator can be used to iterate through an ArrayList, whereas a for or while loop can be used to iterate through an Array.

    4. The length of an ArrayList is provided by the .size() method whereas .length is needed to determine the length of an array.

    5. Elements can be added to an ArrayList using .add() and must be inserted using assignment for an Array.

    6. An array can be multi-dimensional, but ArrayLists must always be one dimensional.

    This is another example of how careful consideration must be given to setting a type before it is declared in Java. Once it is clear what the requirements of the array are, which requires some foresight, a choice can be made between Array or ArrayList.

  5. Basics of the Java Memeory Model

    14 September 2015

    Why Java functions the way it does becomes clearer once the way the Java memory model is understood. I’m sure I’ve only scratched the surface but here is a little of what I have learnt so far.

    The memory model dictates how the virtual memory model interacts with the computer memory (RAM).

    The Java memory model splits memory used by a Java application between thread stacks and the heap. Which of these is used depends on the type defined. Primitive types are stored on the thread stacks and objects or references to those objects are stored on the heap stack.

    If there are multiple variables representing primitive types, these can be stored as separate items on different thread stacks as copies of the original type.

    Using primitive types increases the speed of an application despite the fact that creating Objects references can be easier to use, so I am going to try and use primitive types when it makes sense to do so.
