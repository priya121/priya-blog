  1. More Recursion

    09 December 2015

    Recursion can be a tricky concept and I know I definitely struggled with it when I first started learning how to code. But it is simply a method calling itself with changing parameters that causes it to recurse. This can often be accomplished by a while loop which actually creates fewer stack frames, but can be a less elegant way of tackling a simple problem.

    So the main driving factor as far as I can see is the clarity of your code and the length of your program (including how much data it needs to analyze and compute).

    This is a great webiste that has a lot of great mini examples and ways to practice recursion.

  2. Functional Programming

    09 December 2015

    I thought I knew what functional programming was but I was wrong! I suppose I need to take one step at a time and I am sure during my apprenticeship I will eventually come to learn a functional language, but there is no harm in knowing the basics. So I thought I would share what I came across.

    A functional language is a language that avoid mutating or changing data. It really helps break a problem down into its component parts. So you would just create new data structures instead of changing ones that already exist.

    Functional programs are required to also be stateless, that is they should perform every task as if they were performing them for the first time. This is a pretty different way of thinking and I have just got my head around object oriented (OO) programming so am glad I will tackle functional programming once I have a bit more practice at writing OO languages.

  3. Intellij

    09 December 2015

    Starting to learn Java, Intellij is really a great tool. I have just migrated to using vim and it is clear I was really using Intellij for a lot of things I took for granted. Despite occassionally being stuck in CAPS LOCK it allowed me to learn how to set up a java project very quickly.

    Java is really verbose and Intellij imports a lot of dependencies and autocompletes a lot and this was really useful, but hid a lot of what was going on and what Java requires to run.

    I am still exploring the benefits of learning how to write Java in pure vim and feel like I am learning a lot. I will report back after I get more comfortable with Java in vim and can properly weigh up the benefits and drawbacks, but if you are starting to learn Java for the first time, I can’t recommend Intellij enough.

  4. Black Box Tests

    10 December 2015

    Testing and knowing what to test can be tough when you begin to code. Starting with a degenerate case is a good approach but sometimes this can lead to testing implementation details that can change as your code evolves. This is where black box testing comes in. Here you only test the public methods that handle the overall functionality and behaviour you want from a piece of code. White box testing is where you test the internals of a class or individual methods that handle small pieces of behaviour that you eventually want to hide as private methods.

    I know I am guilty of writing simple and small tests that test individual functions and I tend not to change these to black box tests even when I know I no longer need them. I am going to keep in mind the advantages of black box testing and try and think of the overall functionality I want to test more going forward.
