Maven

15 December 2015

A lot of the building of a Java project is hidden when you are using Intellij. In order to get a project running from the command line it is necessary to use a build tool and Maven is one of these tools. It can build projects in Ruby, Scala and C# but is mainly used to build Java.

When a language is built, this simply means to compile the program into an executable format (taking the source files and compiling them into object files.) A compiler takes the source code written in a language and converts it into another computer language (often binary).

Maven works by using an XML file that describes the project and all the dependencies, build order and plug-ins.

Maven projects are configured using a POM or Project Object Model and these are stroed in a pom.xml fie. Here is the wikipedia example:

<project>
<!-- model version is always 4.0.0 for Maven 2.x POMs -->
<modelVersion>4.0.0</modelVersion>

<!-- project coordinates, i.e. a group of values which
uniquely identify this project -->

<groupId>com.mycompany.app</groupId>
<artifactId>my-app</artifactId>
<version>1.0</version>

<!-- library dependencies -->

<dependencies>
<dependency>

<!-- coordinates of the required library -->

<groupId>junit</groupId>
<artifactId>junit</artifactId>
<version>3.8.1</version>

<!-- this dependency is only used for running and compiling tests -->

<scope>test</scope>

</dependency>
</dependencies>
</project>

When you write a Maven project the structure you need is:

//

By running the command: mvn package all the java files are compiled, all tests run and it packages all deliverable code and resources into target/my-app-1.0.jar.

The command I used to run my maven project (MineSweeper) was: mvn exec:java -Dexec.mainClass=”com.example.Main” in which com.expample are packages you may have introduced in your class structure.
