  1. Pull Requests on Git

    28 September 2015

    Creating a git pull request allows a developer to ask others to review their code when a new feature has been added, a change has been made or they want other developers to review their code for any other reason.

    In order to do this the following steps are necessary:

    In order to work on an exisiting project, a fork of the exisiting project repository needs to be made. Once this is done a copy of the project will have been made.

    Then a clone of the forked repository must be made, this will create a new working copy of the project on a local machine.

    Before creating a new feature, a new branch for the feature has to be created and any changes must be added and commited to this branch.

    Once these changes have been made it is possible to push these changes to the copy of the repository, (not the main repository) by a git push.

    After this has been done, a pull request can be made through the relevant copied repository page.

    Creating pull requests is useful when working in a team and it encourages collaboration.

  2. Static Methods

    28 September 2015

    Static methods in java are used when the method needs to be called without creating an object of the class.

    An instance variable needs an object to be created for it to be called whereas a static method does not. That is why most main classes in java must be static.

    Static methods typically take all their data from their parameters and create something from those parameters. They also typically have no reference to variables, so can handle generic calculations well.

    It is possible to make many methods static, but this is bad practice as it is unclear as to what the responsibility of the class is when many methods can be called on it that are not necssarily related to what it does. A static method called on a class is also not polymorpic so this means that it is not possible to override the behavior set by a static method.

    So it is important to think carefully when creating a static method and avoid using them unless necessary.

  3. Arrays vs ArrayLists in Java

    28 September 2015

    Difference between an Array List and Array in Java

    In Java Arrays and ArrayLists are differentiated in some important aspects:

    1. An Array is fixed in length, whereas the ArrayList can be added to and is dynamic in size.

    2. ArrayLists cannot contain primitive data types whereas an Array can contain both Object references and primitive types.

    3. .iterator can be used to iterate through an ArrayList, whereas a for or while loop can be used to iterate through an Array.

    4. The length of an ArrayList is provided by the .size() method whereas .length is needed to determine the length of an array.

    5. Elements can be added to an ArrayList using .add() and must be inserted using assignment for an Array.

    6. An array can be multi-dimensional, but ArrayLists must always be one dimensional.

    This is another example of how careful consideration must be given to setting a type before it is declared in Java. Once it is clear what the requirements of the array are, which requires some foresight, a choice can be made between Array or ArrayList.

  4. Creating Stories

    28 September 2015

    Starting a program or application from scratch can be daunting, but it is also exciting. What helps clarify the direction of where you need to start is a series of stories. The first few stories are important as they can help guide the rest of the application.

    It is important to start with the most degenerate case and then test the code from there. That is why it is not always useful to have too much planned out for the later stages of the code as it is quite likely that it will change.

    I have started on some stories for a minesweeper game and the most degenerate case for that is the return of an exploded mine (that I will represent as a 1) when the relevant index (depicted by a coordinate) will change.

    I have the next few stories written down and the later they get the more abstarct they become. But this is a useful exercise that I hope I will get better at, becuase so often once you have figured out how to do something, actually doing it and putting it into code is much easier.

  5. Primitive and Object Types in Java

    28 September 2015

    Java is a statically typed language, which means that every variable and every expression has a type that must be known at compile time.

    Types limit the values a variable can hold or an expression can produce, but also help reduce errors in the long run.

    So in Java, data is stored in the form of data types and every variable has a data type.

    The data types in Java are split into two :-

    primitive data types and object types/references

    PRIMITIVE TYPES

    The Java memory model specifies how the Java virtual machine works with the computer memory (RAM).

    The value of a variable stored as a primitive type is typically stored in the memory location assigned to that variable on the thread stack

    There are 8–predefined primitive by the language, here they are in order of increasing sizes and they are limited to those sizes:

    “primitive” means a fundamental component that is used to create other, larger parts

    When you create variable stored as a primitive type instead of creating an object on the heap, Java creates a kind of automatic variable that is not a reference. This variable holds the value directly and is placed on the stack making access more efficient.

    int x = 3; > this means at the memory location of x, 3 will be stored there

    Primitive types also come in versions that are objects. Just capitalised and these have their own set of methods that can be called on them.

    OBJECT TYPES

    A variable of an object type is also called a reference. This is because the variable doesn’t contain the object, but contains a reference pointing to the object. These are stored in the heap.

    The different object types in java are :

    interface types, class types and array types

    In order to create an object reference you must instantiate an object first so that you can manipulate it. So you don’t manipulate the object directly, but use a reference to the object. The reference to the object is like a remote control to the actual object the TV. So you can access the object by calling certain methods on the reference.

    When you create an object in Java, it is stored on the heap. There is generally no limit to memory size in this area. To create an object on the heap, you would use the new keyword like this:-

    eg. Integer myInteger = new Integer(45);

    MEMORY MODEL

    Primitive types and object types are stored within the Java Memory Model

    A local variable (one not accessible by the class) may be of a primitive type, in which case it is kept on the thread stack.

    A local variable may also be a reference to an object. In that case the reference (the local variable) is stored on the thread stack, but the object itself if stored on the heap.

    An object may contain methods and these methods may contain local variables. These local variables are also stored on the thread stack.

    The member variables of an object however, are stored on the heap along with the object itself. That is true both when the member variable is of a primitive type, and if it is a reference to an object. The heap contains all objects created in your Java application, regardless of what thread created the object.

    AUTOBOXING

    Before Java 5 you had to call a method on an object version of a primitive type in order to get the value

    But with Java 5, Java automatically converts a primitive type like int into corresponding wrapper class object and unwraps an object if a primitive type is required

    eg. ArrayList intList = new ArrayList(); intList.add(1); //autoboxing–primitive to object

    int number = intList.get(0); // unboxing–object to primitive

    Need to be careful, if you try to convert null to a primitive value you will get a NullPointerException

    Integer myInteger = null; int myInt = myInteger;

    Using operators and comparing values, and can also lead to unnecessary memory use.
