  2. Loops in Java

    08 September 2015

    Loops in Java are useful even in the most simple program and contain their contents within curly braces. The two most common ones I have used so far are the for loop;

    eg. for( int i = 0; i < 5; i++ )

    and the while loop;

    eg. while ( i < number.length )

    Keeping track of the variables and loops can again be clarified through the use of an instruction tab.

    It is also possible to escape out of a loop if a certain condition is met using a label just before the beginning of the loop and a break after the end of the loop.

  3. Method Signatures in Java

    15 September 2015

    The main class in java must be present in the application at least once for it to be able to run and has a pretty standard signature.

    Methods in java have to have certain elements in order for the Java package to be able to collect the parts of the program togethr and process them in a logical way.

    Firstly a method must be public or private. Public means that it is available across packages whereas private means it is only accessible within the current package. Then if a method is static(that is, it is not called by Objects within the class) this must also be part of the method signature.

    These are followed by what the method signature will return (String, Integer, String[], void etc.) and the method name.

    In Java if a method takes an arguement, the type of arguement must also be declared within parentheses, this is true even if the method takes no arguement with the parentheses being left blank.

    Method signatures in Java seem complicated at first, but I have been told that this makes things clearer in the long term, so far most of my programs and tasks have been small, but I will report back when I know if this is true.

  4. The Single Responsibility Principle

    07 September 2015

    The single responsibility principle means that a class should only have one reason to change.

    The idea is that each class should only have one role or handle one piece of behaviour. This does not mean that each class should contain one method, but if when describing a class behaviour, the word and is used then it is quite likely that the class is doing too much and the additional behaviour should be split out.

    If the SRP is implemented well, then the design of the program will be more flexible and easier to change.

    This is because if a class changes then only the class that handles that behaviour will have to change and the code will not be too coupled–meaning that one small change that must be made leads to numerous other things being forced to change.

  5. Stacks

    18 September 2015

    Stacks are a pretty fundamental concept in CS. They represent how data is allocated to memory at a basic level. A stack area on computer memory is of a fixed origin and variable size. The two basic operations that are used most frequently on a stack are inserting an item and removing an item. A stack uses the concept of last in first out (LIFO) as opposed to a queue which uses the concept of (FIFO). In Java it is possible to create a stack and call certain predefined methods on it like push, pop, peek, search and size. Like most Java lists a stack has to contain matching objects within it that must be declared when it is first created.
