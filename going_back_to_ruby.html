Back to Ruby

25 November 2015

The other day I helped a student apprentice with Ruby and it was strange going back to a dynamic language after focusing on Java for so long. Although it felt good not having the restrictions that Java has, I realised that it takes a little longer to understand what is going on if the names are not clear, and it is hard to keep track of what a variable represents as they can change back and forth so quickly. This is something that I should probably keep in mind when I go back to Ruby. Make the names incredibly clear and representative of what they are, or things can get confusing. I know I started with Ruby, but I think I might just prefer statically typed languages for now...
